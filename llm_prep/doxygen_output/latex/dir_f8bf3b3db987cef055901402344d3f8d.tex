\doxysubsection{Detailed Description}
\hypertarget{dir_f8bf3b3db987cef055901402344d3f8d_autotoc_md1339}{}\doxysubsection{\texorpdfstring{DOS EXE Unpackers}{DOS EXE Unpackers}}\label{dir_f8bf3b3db987cef055901402344d3f8d_autotoc_md1339}
Collection of simple tools for unpacking common DOS executable packers (LZEXE, EXEPACK, PKLITE). These are useful for reverse engineering old MS-\/\+DOS games and tools before disassembly/analysis.

These are integrated as source code so the repo stays self-\/contained.\hypertarget{dir_f8bf3b3db987cef055901402344d3f8d_autotoc_md1340}{}\doxysubsubsection{\texorpdfstring{Build Instructions}{Build Instructions}}\label{dir_f8bf3b3db987cef055901402344d3f8d_autotoc_md1340}
You need gcc (standard on most Linux distros).


\begin{DoxyCode}{0}
\DoxyCodeLine{cd\ tools/dos\_unpackers}
\DoxyCodeLine{make\ \ \ \ \ \ \ \ \ \ \#\ builds\ unlzexe,\ unexepack,\ depklite}

\end{DoxyCode}


Binaries will appear in this directory. Add it to your PATH or call them with relative paths from AXE agents/scripts.\hypertarget{dir_f8bf3b3db987cef055901402344d3f8d_autotoc_md1341}{}\doxysubsubsection{\texorpdfstring{Usage Examples}{Usage Examples}}\label{dir_f8bf3b3db987cef055901402344d3f8d_autotoc_md1341}

\begin{DoxyItemize}
\item ./unlzexe packed.\+exe → creates unpacked version (usually with -\/unlzexe suffix)
\item ./unexepack packed.\+exe → creates "{}unpacked"{} by default
\item ./depklite packed.\+exe → creates packed.\+exe.\+dep (extracted data)
\end{DoxyItemize}\hypertarget{dir_f8bf3b3db987cef055901402344d3f8d_autotoc_md1342}{}\doxysubsubsection{\texorpdfstring{Attributions \& Credits}{Attributions \& Credits}}\label{dir_f8bf3b3db987cef055901402344d3f8d_autotoc_md1342}
