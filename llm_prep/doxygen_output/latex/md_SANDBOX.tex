AXE now supports a {\bfseries{bubblewrap-\/based sandbox}} that provides secure, isolated command execution. This replaces the traditional whitelist-\/based security model with a {\bfseries{default-\/allow}} approach inside a hardened sandbox environment.\hypertarget{md_SANDBOX_autotoc_md1028}{}\doxysection{\texorpdfstring{Key Features}{Key Features}}\label{md_SANDBOX_autotoc_md1028}
\hypertarget{md_SANDBOX_autotoc_md1029}{}\doxysubsection{\texorpdfstring{1. Default-\/\+Allow Execution}{1. Default-\/\+Allow Execution}}\label{md_SANDBOX_autotoc_md1029}

\begin{DoxyItemize}
\item {\bfseries{Inside sandbox}}\+: All tools are allowed by default
\item {\bfseries{Optional blacklist}}\+: Block specific dangerous commands if needed
\item {\bfseries{Host protection}}\+: Host filesystem remains protected via namespace isolation
\end{DoxyItemize}\hypertarget{md_SANDBOX_autotoc_md1030}{}\doxysubsection{\texorpdfstring{2. Linux Namespace Isolation}{2. Linux Namespace Isolation}}\label{md_SANDBOX_autotoc_md1030}

\begin{DoxyItemize}
\item {\bfseries{User namespace}}\+: Commands run as root inside sandbox, unprivileged outside
\item {\bfseries{Mount namespace}}\+: Private filesystem view, only explicitly bound paths visible
\item {\bfseries{PID namespace}}\+: Process isolation -\/ sandbox can\textquotesingle{}t see host processes
\item {\bfseries{Network namespace}}\+: Optional network isolation (disabled by default for convenience)
\item {\bfseries{IPC/\+UTS/\+CGroup namespaces}}\+: Additional isolation layers
\end{DoxyItemize}\hypertarget{md_SANDBOX_autotoc_md1031}{}\doxysubsection{\texorpdfstring{3. Filesystem Access Control}{3. Filesystem Access Control}}\label{md_SANDBOX_autotoc_md1031}

\begin{DoxyItemize}
\item {\bfseries{Workspace writable}}\+: Project directory is read-\/write inside sandbox
\item {\bfseries{System paths read-\/only}}\+: {\ttfamily /usr}, {\ttfamily /lib}, {\ttfamily /bin}, {\ttfamily /etc} exposed read-\/only for tools
\item {\bfseries{Hidden paths}}\+: {\ttfamily \texorpdfstring{$\sim$}{\string~}/.ssh}, {\ttfamily /root}, etc. completely inaccessible from sandbox

\hypertarget{md_SANDBOX_autotoc_md1041}{}\doxysubsection{\texorpdfstring{What Sandbox Protects}{What Sandbox Protects}}\label{md_SANDBOX_autotoc_md1041}
✅ {\bfseries{Host filesystem}} -\/ Only bound paths accessible, rest hidden ~\newline
 ✅ {\bfseries{System processes}} -\/ PID isolation prevents process enumeration ~\newline
 ✅ {\bfseries{User credentials}} -\/ {\ttfamily \texorpdfstring{$\sim$}{\string~}/.ssh}, {\ttfamily \texorpdfstring{$\sim$}{\string~}/.gnupg} completely hidden ~\newline
 ✅ {\bfseries{Root access}} -\/ Commands run unprivileged despite appearing as root inside ~\newline
 ✅ {\bfseries{TTY attacks}} -\/ {\ttfamily new\+\_\+session} prevents terminal control escape ~\newline
\hypertarget{md_SANDBOX_autotoc_md1042}{}\doxysubsection{\texorpdfstring{What Sandbox Doesn\textquotesingle{}t Protect (by design)}{What Sandbox Doesn\textquotesingle{}t Protect (by design)}}\label{md_SANDBOX_autotoc_md1042}
⚠️ {\bfseries{Workspace contents}} -\/ Workspace is writable, code can modify project files ~\newline
 ⚠️ {\bfseries{Network access}} -\/ Default config allows network (set {\ttfamily network\+: true} to isolate) ~\newline
 ⚠️ {\bfseries{CPU/memory}} -\/ No resource limits (use cgroups separately if needed) ~\newline
