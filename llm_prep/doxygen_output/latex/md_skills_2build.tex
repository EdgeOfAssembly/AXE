\hypertarget{md_skills_2build_autotoc_md1166}{}\doxysection{\texorpdfstring{Overview}{Overview}}\label{md_skills_2build_autotoc_md1166}
This skill provides guidelines for using silent/quiet compilation and configuration flags across multiple build systems. By defaulting to silent builds, agents can achieve {\bfseries{50-\/90\% token savings}} in log-\/related output while preserving build functionality.\hypertarget{md_skills_2build_autotoc_md1167}{}\doxysection{\texorpdfstring{Why Silent Builds?}{Why Silent Builds?}}\label{md_skills_2build_autotoc_md1167}
Build systems can generate verbose output that consumes hundreds or thousands of tokens per invocation. When build logs are included in LLM prompts or agent collaborations, this verbosity\+:


\begin{DoxyItemize}
\item {\bfseries{Wastes tokens}}\+: Verbose compiler output rarely contains actionable information
\item {\bfseries{Obscures errors}}\+: Critical error messages get lost in noise
\item {\bfseries{Slows iteration}}\+: More output means slower agent processing
\item {\bfseries{Costs money}}\+: Every token sent to an LLM has a cost
\end{DoxyItemize}

Silent builds solve this by\+:
\begin{DoxyItemize}
\item Showing only warnings, errors, and critical output
\item Preserving the ability to re-\/run with verbose output when debugging
\item Reducing log size by 50-\/90\% in typical projects
\end{DoxyItemize}\hypertarget{md_skills_2build_autotoc_md1168}{}\doxysection{\texorpdfstring{Token Savings Estimates}{Token Savings Estimates}}\label{md_skills_2build_autotoc_md1168}

\begin{DoxyItemize}
\item {\bfseries{Token efficiency}}\+: Save 50-\/90\% on build log tokens

\item {\bfseries{Cost}}\+: Reduce API costs by minimizing log verbosity
\end{DoxyItemize}
